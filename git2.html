A Gitről.

A Git egy nyílt forráskódú, elosztott verziókezelő szoftver, vagy másképpen egy szoftverforráskód-kezelő rendszer, amely a sebességre helyezi a hangsúlyt. A Gitet eredetileg Linus Torvalds fejlesztette ki a Linux kernel fejlesztéséhez. Minden Git munkamásolat egy teljes értékű repository teljes verziótörténettel és teljes revíziókövetési lehetőséggel, amely nem függ a hálózat elérésétől vagy központi szervertől.

 

A Githubról.

A GitHub, Inc. egy egyesült államokbeli nemzetközi vállalat, amely a Git segítségével szoftverfejlesztési verziókövetés-szolgáltatást nyújt. 2018-ban a Microsoft leányvállalata lett 7,5 milliárd dollárért. Saját funkcióin felül a Git elosztott verziókövetését és forráskódkezelését (SCM) teszi elérhetővé. Hozzáférés-kezelést és számos együttműködési funkciót nyújt, mint például bugkövetés, szolgáltatáslekérés, feladatkezelés, valamint wikiket minden projekthez.

 

Git parancsok.

  1. Felhasználónév, email cím.
    • git config --global user.name "Github-on megadott név"
    • git config --global user.email "Github-on regisztrált emailcím"
  2. Repository létrehozás.
    • mkdir repository-name
    • cd repository-name
    • git init
  3. Távoli kapcsolat beállítása feltöltéshez.
    • git remote add origin repository-link (ez a link githubon code-ból elérhető közvetlen link)
    • git remote -v (a kapcsolat lekérdezése)
    • git fetch origin (távoli kapcsolat frissítése)
    • git branch –a (az összes branch lekérdezése)
  4. Fájl hozzáadása az átmeneti tárolóhoz. Pirossal jelzi azokat, amik nem kerültek be, ellenkező esetben zölddel azokat amik bekerültek.
    • git add file-name (egyetlen fájl hozzáadása)
    • git add . (minden az aktuális mappában, vagy annak almappáiban található fájl hozzáadása)
  5. Státusz lekérdezése.
    • git status
  6. Átmeneti tárolóból a repoba való áttöltés
    • git commit -m "Kötelező megjegyzést tenni"
  7. Eddigi verzió megtekintése
    • git log
  8. Fájl és verzió feltöltése távoli tárhelyre
    • git remote –v (kapcsolatot ellenőrizzük)
    • git push origin master
  9. Távoli tárhely adatainak letöltése, helyi verzió frissítése
    • git pull origin master
